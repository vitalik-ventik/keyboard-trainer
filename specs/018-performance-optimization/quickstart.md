# Quickstart: Оптимізація продуктивності для слабких ноутбуків

**Feature**: 018-performance-optimization
**Date**: 2026-07-20

---

## Передумови

1. Сучасний браузер: Chrome 120+, Edge 120+ або Firefox 128+
2. Локальний статичний сервер (для ES6-модулів):
   ```powershell
   .\start-http.cmd
   ```
3. Відкрити `http://localhost:8000` (або порт, вказаний у start-http.cmd)
4. Ноутбук з інтегрованою графікою (Intel UHD 620 / AMD Vega 3) — для перевірки FPS

---

## Крок 1: Конвертація victory.wav → victory.mp3

```powershell
# Використовуючи ffmpeg (має бути встановлений):
ffmpeg -i sounds/victory.wav -c:a libmp3lame -b:a 192k sounds/victory.mp3

# Перевірити розмір:
Get-Item sounds/victory.mp3 | Select-Object Length
# Очікувано: ~260 KB (було 4.3 MB)

# Видалити старий файл:
Remove-Item sounds/victory.wav
```

**Очікуваний результат**: `sounds/victory.mp3` існує, `sounds/victory.wav` відсутній. Загальний розмір `sounds/` зменшився з ~4.78 MB до ~0.73 MB.

## Крок 2: Верифікація аудіо-завантаження

1. Відкрити `http://localhost:8000`
2. Відкрити DevTools (F12) → Console
3. Спостерігати: екран завантаження має завершитися без помилок
4. Перевірити: жодних `console.warn` про `victory.wav`
5. Запустити рівень 1, пройти до перемоги — звук victory має відтворитися

**Очікуваний результат**: Індикатор завантаження заповнюється до 100% без затримок. Звук victory відтворюється при перемозі.

## Крок 3: Вимірювання FPS до/після оптимізації

### Метод вимірювання

```javascript
// У консолі DevTools під час гри:
let fpsFrames = 0;
let fpsLast = performance.now();
function fpsMeter(now) {
    fpsFrames++;
    if (now - fpsLast >= 1000) {
        console.log('FPS:', fpsFrames);
        fpsFrames = 0;
        fpsLast = now;
    }
    requestAnimationFrame(fpsMeter);
}
requestAnimationFrame(fpsMeter);
```

### Тестові рівні для вимірювання

| Рівень | Тема фону | Складність рендеру | Цільовий FPS |
|--------|-----------|-------------------|-------------|
| 1 | cyber_grid | Низька | 60 |
| 18 | rooftop_grid | Середня | 55-60 |
| 29 | nebula_drift | Висока | 50-60 |
| 31 | inferno_core | Дуже висока (бос) | 50-60 |

**Очікуваний результат**: На Intel UHD 620 при 1366x768 усі рівні показують стабільні 50-60 FPS (жоден < 50).

## Крок 4: Перевірка візуальної ідентичності

### Фони
1. Зробити скріншоти кожного рівня у старій версії (до оптимізації)
2. Зробити скріншоти тих самих рівнів у новій версії
3. Порівняти попарно — кольори, позиції, прозорість, glow-ефекти мають збігатися

### Скіни гравця
1. Відкрити селектор скінів у головному меню
2. Перевірити візуально всі 31 скін — кожен має зберегти neon glow стиль
3. Особливу увагу: `meteor` (має trail), `demon_lord` (має horns + shadowBlur 25)

### Клавіатура
1. Запустити будь-який рівень
2. Підсвітка цільової літери (зелено-жовтий glow) — візуально ідентична
3. Натиснути помилкову літеру — червона підсвітка з'являється і згасає за 350ms так само

**Очікуваний результат**: Усі скріншоти до/після ідентичні.

## Крок 5: Перевірка граничних кейсів

### Ресайз вікна
1. Запустити будь-який рівень
2. Змінити розмір вікна (Win + ← / Win + → для snap layouts)
3. Фон і клавіатура мають оновитися без витоку пам'яті
4. Перевірити в Task Manager: споживання пам'яті вкладки не зростає безмежно

### Приховання/показ вкладки
1. Перемкнутися на іншу вкладку на 5+ секунд
2. Повернутися — гра має продовжитися без стрибка фізики
3. Жодних `console.warn` про `dt` або пропущені кадри

### Відсутність аудіо-файлу
1. Тимчасово перейменувати `sounds/click.wav` → `sounds/click.wav.bak`
2. Відкрити застосунок — завантаження має завершитися
3. Кнопки мають працювати без звуку (без фатальної помилки)
4. У консолі — `console.warn` про відсутність файлу
5. Повернути назву файлу назад

### Демо-режим у меню
1. Відкрити головне меню — демо-фон має анімуватися плавно
2. Змінити налаштування (difficulty, speed) — демо перестворюється без помилок
3. FPS у демо-режимі — стабільні 60

### Швидкий набір на клавіатурі
1. Запустити рівень 31 (speed x1.25 FAST, 450×1.25 = 562 px/s)
2. Швидко натискати правильні/помилкові літери
3. Підсвітка клавіатури оновлюється без візуального відставання

**Очікуваний результат**: Усі граничні кейси проходять без помилок у консолі.

## Крок 6: Профілювання (Performance tab)

1. Відкрити Chrome DevTools → Performance tab
2. Натиснути Record, пограти 10-15 секунд на рівні 31, Stop
3. Проаналізувати:
   - **Main thread**: жоден frame не перевищує 16.6ms
   - **Rasterizer**: не більше 30% від budget кадру
   - **GC**: мінімальна активність (< 1ms на кадр)
   - **Function calls**: `drawImage` замість десятків `fillRect`/`stroke` для фону
4. У розділі Bottom-Up знайти `shadowBlur` — має бути 0 викликів

**Очікуваний результат**: Жодного виклику `shadowBlur` у гарячому шляху. `drawImage` для кешу фону займає < 0.5ms.
