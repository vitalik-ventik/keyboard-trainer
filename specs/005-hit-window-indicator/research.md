# Research: Візуальна індикація вікна стрибка на лінії землі

**Feature**: 005-hit-window-indicator
**Phase**: 0 — Outline & Research
**Date**: 2026-07-18

## Research Questions

Технічний контекст не містить NEEDS CLARIFICATION маркерів — усі деталі реалізації надані в описі фічі користувачем. Дослідження спрямоване на верифікацію наявних механізмів Engine, які використовуватиме новий метод.

## R1: Наявність `okPx` і `perfectPx`

**Питання**: Чи справді поля `okPx` і `perfectPx` вже існують в Engine і коректно розраховуються?

**Дослідження**: Перевірено `js/engine.js`.

- `this.okPx = this.level.speed * windows.okTime;` (рядок 330)
- `this.perfectPx = this.level.speed * windows.perfectTime;` (рядок 331)
- Розраховуються в конструкторі `Engine` на основі швидкості рівня та `hitWindowTimes(levelId)`
- Використовуються в логіці стрибка: `consumeJumpBuffer()` (рядок 413), `handleLetter()` (рядки 484, 497), демо-автогра (рядок 586)

**Decision**: Поля існують і коректні. Використовувати `this.okPx` для зовнішньої зони, `this.perfectPx` для внутрішньої.

**Rationale**: Специфікація вимагає не додавати нових полів Engine (FR-008).

## R2: Механізм визначення активного шипа попереду

**Питання**: Як визначити, чи є активний шип попереду (state === "ahead")?

**Дослідження**: Клас Engine має метод `nearestAheadSpike()` (рядки 369–376):

```js
nearestAheadSpike() {
    for (const spike of this.spikes) {
        if (spike.state === "ahead" && spike.x + SPIKE_W / 2 >= this.player.x) {
            return spike;
        }
    }
    return null;
}
```

Повертає `null`, якщо активних шипів попереду немає.

**Decision**: Викликати `this.nearestAheadSpike()` у `renderHitWindow()`. Якщо повертає `null` — не малювати індикатор.

**Rationale**: Перевикористання існуючого методу без дублювання логіки, що зменшує ризик розсинхронізації.

---

## R3: Доступ до `time` у методі рендера

**Питання**: Чи доступний параметр `time` (для пульсації зеленої зони) у контексті `renderHitWindow`?

**Дослідження**: Метод `render(ctx, W, H, time)` (рядок 646) отримує `time` як останній параметр. Під-методи рендера (`renderBackground`, `renderGround`, `renderSpikes`, тощо) не отримують `time` безпосередньо, але `renderBackground` використовує `time` через `renderBossBackground` (рядок 669).

**Decision**: Передавати `time` четвертим параметром у `renderHitWindow(ctx, W, groundY, anchorX, time)`. Для пульсації використовувати `Math.sin(time * 6)` як зазначено в технічних вимогах.

**Rationale**: `time` потрібен лише для анімації пульсації зеленої зони (FR-004). Передача параметром — найпростіший спосіб без додавання полів.

## R4: Умова «кубик живий»

**Питання**: Як перевірити, чи кубик живий у контексті рендера?

**Дослідження**: `this.player.alive` (рядок 348) — булеве поле, що встановлюється в `false` при вибуху (`explode()`, рядок 444). Метод `renderPlayer` (рядок 869) також використовує цю перевірку: `if (!this.player.alive) { return; }`.

**Decision**: Використовувати `this.player.alive` як першу перевірку в `renderHitWindow`. Якщо `false` — не малювати.

**Rationale**: Відповідає FR-002(a) та поведінці інших методів рендера.

---

## R5: Місце виклику в `render()`

**Питання**: Де саме вставити виклик `renderHitWindow` у ланцюжку рендера?

**Дослідження**: Поточний порядок у `render()` (рядки 646–660):

```
renderBackground → renderGround → renderFinish → renderSpikes → renderPlayer → renderParticles → renderProgressBar
```

Технічні вимоги: після `renderGround()`, перед `renderSpikes()`.

**Decision**: Викликати `this.renderHitWindow(ctx, W, groundY, anchorX, time)` між `this.renderGround(...)` і `this.renderFinish(...)`.

**Rationale**: Індикатор малюється на лінії землі — має бути поверх землі, але під шипами, щоб шипи з літерами були добре видимі. Позиція після землі й перед шипами забезпечує правильний z-order. Виклик до `renderFinish` також прийнятний — фінішний портал і так далеко попереду.

## R6: Візуальні параметри індикатора

**Питання**: Які точні значення кольорів і тіней використовувати?

**Дослідження**: Усі параметри надані в технічних вимогах:

| Параметр | OK Zone (зовнішня) | Perfect Zone (внутрішня) |
|----------|-------------------|--------------------------|
| Колір заливки | `rgba(0, 246, 255, 0.15)` | `rgba(57, 255, 136, 0.35)` |
| `shadowBlur` | 6 | 10 |
| `shadowColor` | `rgba(0, 246, 255, 0.15)` (колір заливки) | `rgba(57, 255, 136, X)` (пульсуюча яскравість) |
| Пульсація | Ні (статична) | Так: `Math.sin(time * 6)` модулює alpha |
| Межа | Чітка, без градієнта | — |

**Decision**: Використовувати точно вказані значення. Для пульсації зеленої зони: `alpha = 0.35 + 0.15 * (Math.sin(time * 6) + 1) / 2`, що дає діапазон прозорості 0.35–0.50.

**Rationale**: Діапазон пульсації 0.35–0.50 забезпечує помітну, але не надто агресивну анімацію. Центрування навколо базового значення 0.35 з амплітудою 0.15 дає природний ефект «дихання».

## R7: Розташування індикатора на землі

**Питання**: Як саме розташувати індикатор відносно лінії землі?

**Дослідження**: Лінія землі проходить на `y = groundY` (рядок 647: `const groundY = H * 0.64`). Індикатор має бути на лінії землі, з невеликою висотою для видимості.

**Decision**: Малювати прямокутник висотою 6 px з центром по вертикалі на `y = groundY`:
- `y = groundY - 3` (верхня межа)
- `height = 6` px

Зовнішня зона: від `anchorX` до `anchorX + this.okPx`.
Внутрішня зона: від `anchorX` до `anchorX + this.perfectPx`.

**Rationale**: Висота 6 px достатня для видимості, але не перекриває шипи (SPIKE_H = 48, які починаються від groundY вгору). Позиція відносно anchorX відповідає FR-001.

## Summary of Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Довжини зон | `this.okPx`, `this.perfectPx` | Існуючі поля Engine (FR-008) |
| Перевірка шипа | `this.nearestAheadSpike()` | Існуючий метод, уникнення дублювання |
| Параметр time | Передати в `renderHitWindow(t)` | Потрібен для пульсації (FR-004) |
| Перевірка кубика | `this.player.alive` | Існуюче поле (FR-002) |
| Місце виклику | Після `renderGround`, перед `renderSpikes` | Правильний z-order |
| Висота індикатора | 6 px, центровано на groundY | Видимість без перекриття шипів |
| Пульсація alpha | `0.35 + 0.15 * (sin(t*6)+1)/2` | Діапазон 0.35–0.50, природне «дихання» |
