# Research: Адаптивна дуга стрибка

**Created**: 2026-07-19  
**Feature**: [spec.md](./spec.md)

## Research Tasks

### RT-1: Валідація формули фізики кинутого тіла

**Decision**: Використовувати формулу `vy = GRAVITY * distance / (2 * effectiveSpeed)`

**Rationale**: Формула виводиться з базової фізики рівномірного руху по горизонталі та рівноприскореного по вертикалі:
- Час польоту: `t_total = 2 * vy / GRAVITY` (вгору й униз симетрично)
- Горизонтальна відстань: `dx = effectiveSpeed * t_total = effectiveSpeed * 2 * vy / GRAVITY`
- Звідси: `vy = GRAVITY * dx / (2 * effectiveSpeed)`

Підстановка `dx = distance = gap + SPIKE_W/2 + SAFE_MARGIN` гарантує, що кубик подолає горизонтальну відстань, рівну сумі: відстань до центра шипа + півширини шипа + посадковий зазор.

**Alternatives considered**:
- Фіксований набір висот (низький/середній/високий стрибок): відхилено як недостатньо плавний.
- Квадратична залежність від відстані: відхилено як нефізичну.

### RT-2: Визначення оптимального SAFE_MARGIN

**Decision**: `SAFE_MARGIN = 25` пікселів

**Rationale**:
- `SPIKE_W = 44`, отже `SPIKE_W/2 + SAFE_MARGIN = 22 + 25 = 47` px посадкового запасу.
- На найшвидшому рівні (speed=450, level 31):
  - `okPx = 450 * 0.28 = 126` px (максимальна дистанція влучання)
  - `max distance = 126 + 22 + 25 = 173` px
  - `vy = 2600 * 173 / (2 * 450) ≈ 500` — помірна швидкість, комфортна дуга.
- На найповільнішому рівні (speed=165, level 1):
  - `okPx = 165 * 0.45 = 74.25` px
  - `max distance = 74.25 + 22 + 25 = 121.25` px
  - `vy = 2600 * 121.25 / (2 * 165) ≈ 955` — високий стрибок на дальній дистанції, природна поведінка.
- Значення 25 px забезпечує візуально комфортний зазор без надмірного «зависання» кубика.

**Alternatives considered**:
- 15 px: надто малий зазор, ризик візуального дотику до шипа.
- 40 px: надто великий зазор, стрибок виглядає неприродньо довгим на близьких дистанціях.

### RT-3: Визначення оптимального MIN_JUMP_VELOCITY

**Decision**: `MIN_JUMP_VELOCITY = 420`

**Rationale**: Поточне `JUMP_VELOCITY = 660` — це фіксована швидкість, яка добре працювала візуально. `MIN_JUMP_VELOCITY = 420` забезпечує:
- Мінімальний час польоту: `2 * 420 / 2600 ≈ 0.323` с
- На найшвидшому рівні (speed=450): `dx = 450 * 0.323 ≈ 145` px — достатньо для перетину шипа.
- На найповільнішому рівні (speed=165): `dx = 165 * 0.323 ≈ 53` px — мінімально достатньо.
- При мінімальній дистанції (наприклад, gap=10): `distance = 10 + 22 + 25 = 57`, обчислена `vy = 2600 * 57 / (2 * 450) ≈ 165` для рівня 31 — це нижче за MIN, тому застосовується `MIN_JUMP_VELOCITY = 420`, що гарантує безпечний стрибок.

**Alternatives considered**:
- 300: занизько — на швидких рівнях кубик може не перетнути шип.
- 660 (поточне JUMP_VELOCITY): зависоко як мінімум — мінімальний стрибок буде надто високим.

### RT-4: Зміна сигнатури методу jump()

**Decision**: Сигнатура `jump(distance, perfect)` — перший параметр `distance` (обчислена відстань), другий — `perfect` (boolean для комбо-лічильника).

**Rationale**:
- У поточному коді `jump(perfect)` приймає лише boolean. Потрібно додати `distance`.
- Виклики:
  - `handleLetter()` — обчислює `distance = (spike.x - player.x) + SPIKE_W/2 + SAFE_MARGIN`, передає в `jump(distance, perfect)`.
  - `consumeJumpBuffer()` — обчислює `distance` на основі актуального `spike.x - player.x` (на момент приземлення), передає в `jump(distance, perfect)`.
  - Демо-блок `update()` — обчислює `distance`, передає в `jump(distance, true)`.
- `player.vy` обчислюється всередині `jump()`:
  ```js
  const computedVy = GRAVITY * distance / (2 * this.effectiveSpeed);
  this.player.vy = Math.max(computedVy, MIN_JUMP_VELOCITY);
  ```

**Alternatives considered**:
- `jump(distance)` без `perfect`: відхилено — порушує існуючу механіку комбо-лічильника.
- Окремий метод `computeJumpVelocity(distance)`: відхилено — зайве ускладнення; обчислення тривіальне.

### RT-5: Аналіз впливу на існуючі рівні

**Decision**: Усі 31 рівень залишаються прохідними. Адаптивна дуга покращує передбачуваність на всіх рівнях.

**Rationale**:
- Формула автоматично масштабує `vy` пропорційно до відстані та обернено пропорційно до швидкості.
- На повільних рівнях (ліга «Базова», speed 165–270): обчислена `vy` для максимальної дистанції в межах 700–1000 — природна дуга.
- На швидких рівнях (ліга «Бос», speed 450): обчислена `vy` для максимальної дистанції ≈ 500 — помірна дуга, бо швидкість компенсує.
- `MIN_JUMP_VELOCITY = 420` гарантує мінімальний захист від падіння на шип навіть при стрибку впритул.
- Демо-режим слугує автоматичним валідатором: якщо демо проходить рівень — рівень прохідний.

### RT-6: Взаємодія з колізіями HARD-режиму

**Decision**: Логіка колізій HARD-режиму НЕ змінюється. Адаптивний стрибок не впливає на перевірку `gap <= SPIKE_W * 0.5 + CUBE_SIZE * 0.4`.

**Rationale**:
- HARD-режим перевіряє, чи кубик не «проґавив» шип (не натиснув вчасно). Це незалежна перевірка, що відбувається до колізії з спрайтом.
- Зміна висоти стрибка не впливає на цю перевірку, оскільки вона базується виключно на горизонтальній відстані `gap`.
- Фізична колізія (спрайт кубика зі спрайтом шипа) також не змінюється — вона перевіряє поточну позицію `player.y < SPIKE_H * 0.72`.

## Summary

Усі дослідницькі питання вирішено. Формула адаптивної дуги валідована фізично. Константи `SAFE_MARGIN = 25` та `MIN_JUMP_VELOCITY = 420` визначені на основі аналізу крайніх випадків (найповільніший і найшвидший рівні). Зміни локалізовані в 4 точках одного файлу без порушення конституції.
