# Quickstart: Динамічні багатошарові Canvas-фони

**Feature**: 013-dynamic-canvas-backgrounds
**Date**: 2026-07-19

## Передумови

- Проект запускається локально через відкриття `index.html` у браузері (або локальний статичний сервер для ES6-модулів)
- Браузер: Chrome 120+, Edge 120+, або Firefox 120+
- Усі файли (`js/`, `css/`, `sounds/`, `music/`) присутні в директорії проекту

## Швидка перевірка (Smoke Test)

### 1. Базовий запуск

```bash
# Відкрити в браузері
start index.html
```

**Очікуваний результат**: Завантажується головне меню з демо-режимом. Фон демо-режиму має новий стиль (неонова сітка/місто тощо).

### 2. Перевірка окремих типів фонів

Відкрийте кожен із цих рівнів через меню вибору рівня:

| Рівень | Очікуваний фон |
|--------|----------------|
| 1-1 «Перші кроки» | 3D перспективна неонова сітка, що рухається на гравця |
| 1-2 «Голосний старт» | 3 шари міських силуетів з різною швидкістю + мерехтіння вікон |
| 1-11 «Шиплячий мікс» | Концентричні рамки, що розходяться від центру з прискоренням |
| 1-12 «Вокальний лабіринт» | Діагональний неоновий дощ (45°) + іскри при досягненні низу |
| 1-14 «Центральні сусіди» | 4 напівпрозорі хвилі, що перетинаються |
| 2-4 «Лівий сектор» | Спадаючі стовпчики «0» та «1» синього кольору |
| 3-1 «Верхній штурм» | Вертикальні стовпчики-еквалайзер по нижньому краю |

### 3. Перевірка системи часток

1. Відкрийте будь-який рівень (наприклад, 1-1)
2. Дочекайтесь першої перешкоди
3. Натисніть правильну клавішу для стрибка

**Очікуваний результат**: У точці контакту з'являється спалах із 15-20 неонових точок, які розлітаються і згасають протягом ~0.5 секунди.

### 4. Перевірка glow-ефекту

1. Відкрийте рівень 1-1 (Кібер-Сітка)
2. Зверніть увагу на лінії сітки та контури перешкод

**Очікуваний результат**: Основні структурні лінії мають м'яке неонове світіння навколо себе.

### 5. Перевірка продуктивності

1. Відкрийте інструменти розробника (F12) → вкладка Performance
2. Запустіть рівень 3-1 (найскладніший візуально)
3. Запишіть 5-секундний відрізок

**Очікуваний результат**: Стабільні 60 FPS (кожен кадр ≤ 16.67ms). Бюджет фону < 8ms на кадр.

### 6. Перевірка граничних випадків

| Дія | Очікувана поведінка |
|-----|---------------------|
| Зміна розміру вікна під час гри | Фон перераховується, без артефактів/розтягування |
| Пауза (Escape) під час гри | Анімація фону зупиняється |
| Відновлення після паузи | Анімація продовжується з того ж стану |
| Game Over | Частки дограють анімацію ~0.5 с, потім екран результатів |
| Швидке подолання 10+ перешкод | Частки не перевищують 50, найстаріші видаляються |

### 7. Перевірка структури файлів

```bash
# Переконатися, що новий модуль існує
ls js/backgrounds.js

# Переконатися, що index.html підключає модуль
grep "backgrounds.js" index.html
```

**Очікуваний результат**: `backgrounds.js` існує та підключений у `index.html`.

## Відомі обмеження

- Рівні 22-24 та 31 (Бос) використовують fallback-теми (starfield/demon), оскільки не входять в основний scope специфікації
- Glow-ефект не застосовується до масових елементів (частки, краплі, символи матриці) для збереження продуктивності
- Для повноцінного тестування потрібно розблокувати всі рівні (можна тимчасово змінити `localStorage`)
