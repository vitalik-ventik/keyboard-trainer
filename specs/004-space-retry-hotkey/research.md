# Research: Заміна гарячої клавіші на Space (004-space-retry-hotkey)

**Date**: 2026-07-18 | **Plan**: [plan.md](./plan.md)

NEEDS CLARIFICATION відсутні. 3 рішення проти реального коду `js/keyboard.js`.

## R1. CONFIRM_CODES = {"Space"} (повна заміна)

- **Decision**: Набір клавіш дії confirm змінюється з
  `{"Enter", "NumpadEnter"}` на `{"Space"}`. Пробіл НЕ виходить із
  `BLOCKED_CODES` — блокування скролу (FR-023/FR-046) діє завжди, а дія
  confirm додається поверх нього.
- **Rationale**: «Поміняти» = замінити (Assumption зі spec). У Пробілу немає
  numpad-аналога, тож набір із одного коду. Контракт `initKeyboardInput`
  (сигнатура, гарантії repeat/модифікаторів) незмінний — main.js не
  редагується (FR-047 успадковується автоматично, включно зі стан-бар'єром).
- **Alternatives considered**:
  - Додати Space до Enter (обидві клавіші) — відхилено: суперечить вимозі
    «поміняти» та Assumption про повну заміну (FR-045).
  - Реалізувати вибір клавіші в main.js — відхилено: політика клавіш належить
    keyboard.js (Принцип II; підтверджено дизайном фічі 002).

## R2. Enter/NumpadEnter → BLOCKED_CODES (блокування без дії)

- **Decision**: `"Enter"` і `"NumpadEnter"` переносяться до `BLOCKED_CODES`:
  `preventDefault` виконується, дія — ні.
- **Rationale**: Гарантія R4 фічі 002 (захист від «фантомної» активації
  сфокусованої DOM-кнопки дефолтним Enter) має пережити заміну клавіші —
  інакше Enter на екрані завершення знову активував би сфокусовану кнопку
  непередбачувано (порушення FR-045/SC-018 у найпідступнішій формі: рестарт
  «нібито від Enter» через фокус). Edge case зі spec зафіксовано саме так.
- **Alternatives considered**:
  - Повністю прибрати обробку Enter — відхилено: повертає баг фокуса,
    усунутий фічею 002.

## R3. Порядок перевірок: модифікатори → блокування → confirm → літери

- **Decision**: У keydown-обробнику перевірка `ctrlKey/altKey/metaKey`
  переноситься НА ПОЧАТОК (до `BLOCKED_CODES.preventDefault`). Далі:
  блокування BLOCKED_CODES → гілка confirm (Space: repeat-фільтр →
  onConfirm) → літери ЙЦУКЕН.
- **Rationale**: Edge case зі spec: Alt+Space (системне меню вікна Windows)
  та Ctrl+Space не повинні перехоплюватися. У поточному коді preventDefault
  для BLOCKED_CODES виконується ДО перевірки модифікаторів — Alt+Space був
  би заблокований. Перенос перевірки модифікаторів на початок виправляє це
  для всіх блокованих клавіш одразу. Побічний ефект безпечний: Ctrl/Alt +
  Стрілки/Tab більше не блокуються, але вони й не скролять сторінку;
  Shift не входить у перелік модифікаторів, тож Shift+Space (скрол угору)
  залишається заблокованим.
- **Alternatives considered**:
  - Особливий випадок лише для Space — відхилено: двоїста політика для
    одного набору BLOCKED_CODES ускладнює інваріанти модуля.
