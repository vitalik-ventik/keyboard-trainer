## Entities

### 1. LevelConfig (Конфігурація рівня)

Статичний масив `LEVELS` з 15 об'єктів. Не змінюється в рантаймі.

| Поле | Тип | Опис | Валідація |
|------|-----|------|-----------|
| `id` | `number` | Ідентифікатор рівня (1–15) | 1 ≤ id ≤ 15, ціле |
| `name` | `string` | Назва рівня | Непорожній рядок |
| `letters` | `string[]` | Масив літер пулу рівня | Підмножина матриці KEYS |
| `newLetters` | `string[]` | Літери, що додаються на цьому рівні | Підмножина letters |
| `speed` | `number` | Швидкість руху (лог. px/с) | 160 ≤ speed ≤ 400 |
| `spikeCount` | `number` | Кількість перешкод на трасі | 12 ≤ spikeCount ≤ 55, ціле |
| `seed` | `number` | Сід для PRNG генерації траси | Ціле > 0 |
| `bgTheme` | `string` | Ідентифікатор візуальної теми | Одне з: `deep_grid`, `city_night`, `synthwave`, `equalizer`, `pulse_grid`, `matrix`, `speed_lines`, `geometry`, `stalactites`, `light_pulse`, `tunnel`, `rain`, `pulse_ripples`, `flame`, `demon` |
| `accentColor` | `string` | Основний неоновий колір рівня | Валідний CSS-колір |
| `rhythmGroups` | `boolean` | Режим ритмічних груп перешкод | true/false |

### 2. Obstacle (Перешкода)

Об'єкт на трасі, що генерується `generateTrack()`.

| Поле | Тип | Опис | Валідація |
|------|-----|------|-----------|
| `x` | `number` | X-позиція на трасі | ≥ 0 |
| `letter` | `string` | Літера для введення | Входить у level.letters |
| `state` | `string` | Стан перешкоди | `"ahead"` \| `"cleared"` \| `"hit"` |
| `type` | `string` | Тип перешкоди | `"spike"` \| `"double_spike"` \| `"saw"` |
| `rotationAngle` | `number` | Кут обертання (тільки для saw) | 0 ≤ angle < 2π |

**State Machine (перешкода)**:
```
ahead ──(правильна літера у вікні)──→ cleared
ahead ──(фізичний контакт)──────────→ hit → explode()
ahead ──(пройдено без стрибка)──────→ cleared (EASY, кінець траси)
```

### 3. PlayerProgress (Прогрес гравця)

Зберігається в `localStorage` під ключем `dfp_save_v1`.

```json
{
  "version": 1,
  "settings": {
    "difficulty": "EASY"
  },
  "progress": {
    "unlocked": 1,
    "levels": {
      "1": { "bestPct": 0, "highScore": 0 },
      "2": { "bestPct": 0, "highScore": 0 },
      ...
      "15": { "bestPct": 0, "highScore": 0 }
    }
  }
}
```

| Поле | Тип | Опис | Валідація |
|------|-----|------|-----------|
| `unlocked` | `number` | Макс. розблокований рівень | 1 ≤ unlocked ≤ 15, ціле |
| `levels.{id}.bestPct` | `number` | Найкращий % проходження | 0 ≤ bestPct ≤ 100, ціле |
| `levels.{id}.highScore` | `number` | Найкращий рахунок | ≥ 0, ціле |

**Правила розблокування**:
- `recordResult(levelId, 100, score)`: `unlocked = max(unlocked, levelId + 1)`, якщо `levelId < 15`
- 100% визначається як досягнення `finishX` (подолання всіх перешкод)

**Міграція**:
- Старі збереження з `unlocked ≤ 6` зберігають значення
- Для відсутніх рівнів (7–15) створюються записи з `{ bestPct: 0, highScore: 0 }`
- `unlocked` клампиться в [1, 15] замість [1, 6]

### 4. WrongKeyError (Стан помилки клавіатури)

In-memory стан, не серіалізується. Зберігається в замиканні `keyboard.js`.

| Поле | Тип | Опис |
|------|-----|------|
| `letter` | `string\|null` | Літера помилково натиснутої клавіші |
| `timestamp` | `number` | Час натискання (performance.now()) |

**Життєвий цикл**:
1. `keydown` → неправильна літера → `{ letter: "П", timestamp: now }`
2. Наступний `keydown` (будь-яка літера) → `{ letter: null, timestamp: 0 }`
3. Рендер: якщо `letter !== null && (now - timestamp) > 350` → скинути в null автоматично

### 5. KeyboardState (Стан клавіатури для рендерингу)

Існуюча структура, що передається в `drawKeyboard`. Розширюється:

| Поле | Тип | Опис |
|------|-----|------|
| `groupLetters` | `string[]` | Літери пулу рівня (cyan) |
| `targetLetter` | `string\|null` | Найближча ціль (lime/yellow) |
| `wrongKeyError` | `{ letter: string\|null, timestamp: number }` | Помилково натиснута (red) |

**Пріоритет кольорів** (при конфлікті літера є і в пулі, і є ціллю, і помилковою — лише один колір):
1. `wrongKeyError.letter` → червоний (найвищий пріоритет)
2. `targetLetter` → жовто-зелений
3. `groupLetters` → блакитний
4. Все інше → сірий

## Entity Relationships

```
LevelConfig ──(1:many)──→ Obstacle (через generateTrack)
Engine ──────(1:1)─────→ LevelConfig (поточний рівень)
Engine ──────(1:many)──→ Obstacle[] (траса)
SaveManager ─(1:1)─────→ PlayerProgress (localStorage)
keyboard.js ─(1:1)─────→ KeyboardState (in-memory, для drawKeyboard)
main.js ─────(1:1)─────→ Engine (ігрова сесія)
main.js ─────(1:1)─────→ KeyboardState (зв'язує Engine з drawKeyboard)
```
