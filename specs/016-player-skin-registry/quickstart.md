# Quickstart: Реєстр скінів гравця (31 рівень)

**Дата**: 2026-07-19

## Призначення

Цей документ описує сценарії ручної валідації для перевірки працездатності системи скінів після імплементації. Не містить коду реалізації — лише кроки перевірки.

## Передумови

- Проект запущено через локальний HTTP-сервер: `start-http.cmd` або `python -m http.server 8080`
- Відкрито `http://localhost:8080` у сучасному браузері (Windows 11)
- `localStorage` очищено для тестування «з нуля» (DevTools → Application → Clear storage)

## Сценарій V1: Базовий скін на першому запуску

1. Відкрити гру → екран MENU → «Грати» → вибрати лігу «Базова» → натиснути рівень 1-1
2. **Очікується**: Кубик синій із білою рамкою та внутрішнім світінням
3. Виконати стрибок (натиснути будь-яку клавішу)
4. **Очікується**: Кубик обертається, скін обертається разом із ним без спотворень

## Сценарій V2: Розблокування скіна при першому проходженні

1. Завершити рівень 1-1 (чекати доки кубик досягне фінішу)
2. **Очікується**: Екран VICTORY із повідомленням «Розблоковано новий скін: Стандартний Неон!»
3. Натиснути «Меню» → вибрати рівень 1-2
4. Завершити рівень 1-2
5. **Очікується**: Повідомлення «Розблоковано новий скін: Кібер-Око!»

## Сценарій V3: Повторне проходження — без сповіщення

1. Вибрати рівень 1-1 (уже пройдений)
2. Завершити рівень
3. **Очікується**: На екрані VICTORY **немає** повідомлення про розблокування скіна

## Сценарій V4: Збереження між сесіями

1. Розблокувати скіни на рівнях 1-1, 1-2, 1-3
2. Закрити браузер повністю
3. Відкрити гру знову
4. Вибрати рівень 1-3
5. **Очікується**: Кубик має скін «Ретро-Геймер» (дві зелені лінії + усмішка)
6. Вибрати рівень 1-5
7. **Очікується**: Кубик має скін «Приціл» (червона мішень)

## Сценарій V5: Усі 31 скін візуально різні

1. Вибрати кожен рівень послідовно (або використовувати DevTools для встановлення `unlocked: 31`)
2. Спостерігати кубик у стані спокою та під час стрибка
3. **Очікується**: Кожен із 31 скіна візуально відрізняється від інших

## Сценарій V6: Скін «Метеор» зі шлейфом

1. Вибрати рівень 3-4 (рівень 28)
2. Виконати стрибок
3. **Очікується**: Позаду кубика видно 2 напівпрозорі копії, що відстають
4. Приземлитися
5. **Очікується**: Шлейф зникає

## Сценарій V7: Динамічні скіни при паузі

1. Вибрати рівень 3-1 (Плазма) або 1-9 (Подвійна Рамка)
2. Спостерігати пульсацію/анімацію ефекту
3. Відкрити DevTools → поставити breakpoint у `requestAnimationFrame`
4. **Очікується**: Ефект призупиняється разом із грою

## Сценарій V8: Відмовостійкість localStorage

1. Відкрити DevTools → Application → Storage
2. Вручну видалити ключ `dfp_save_v1`
3. Перезавантажити сторінку
4. **Очікується**: Гра запускається без помилок, кубик має базовий скін
5. Пройти рівень 1-1
6. **Очікується**: Дані зберігаються в localStorage, скін розблоковано

## Сценарій V9: Продуктивність (FPS)

1. Відкрити рівень 5-1 (найскладніший скін «ЛОРД ДЕМОНІВ»)
2. Увімкнути FPS-лічильник у браузері (або візуально оцінити плавність)
3. **Очікується**: 60 FPS стабільно, без помітних просідань
4. Повторити для рівнів 3-4 (Метеор) та 3-1 (Плазма)

## Сценарій V10: Сумісність з існуючими збереженнями

1. Якщо є старі збереження (з попередньої версії без `unlockedSkins`):
2. Завантажити гру
3. **Очікується**: Гра запускається, помилок у консолі немає
4. Пройти новий рівень
5. **Очікується**: `unlockedSkins` додається до збереження
