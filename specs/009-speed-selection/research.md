# Research: Вибір швидкості гри (Slow / Normal / Fast)

## Проведені дослідження

Spec містить вичерпні технічні вимоги. Усі аспекти визначені однозначно,
[NEEDS CLARIFICATION] маркери відсутні. Жодних додаткових досліджень не
потрібно.

## Підтверджені рішення

### Коефіцієнти швидкості
- **Slow**: 0.75 — затверджено користувачем
- **Normal**: 1.0 — базове значення (зворотна сумісність)
- **Fast**: 1.25 — затверджено користувачем

### Архітектурний підхід
- **Рішення**: Clone+multiply — Engine створює поверхневу копію об'єкта рівня та
  замінює `level.speed` на добуток оригіналу та множника
- **Обґрунтування**: Мінімальне втручання — всі залежні величини (spawn distance,
  player.x += speed*dt, hit window px, saw rotation) автоматично використовують
  скореговану швидкість без змін у логіці рушія
- **Альтернативи**: Додавання множника в кожне місце використання `level.speed`
  — відхилено через дублювання та ризик пропустити одне з місць

### Схема кольорів кнопок
- **SLOW**: блакитний неон (як LARGE в hit window)
- **NORMAL**: зелений неон (як EASY / NORMAL в інших налаштуваннях)
- **FAST**: червоний/рожевий неон (як HARD в складності)

### Поведінка під час гри
- **Рішення**: Швидкість фіксується в момент створення Engine; зміна
  налаштування не впливає на поточний рівень
- **Обґрунтування**: Engine — не реактивний об'єкт; зміна speed під час гри
  потребувала б ре-ініціалізації траси, що неможливо без перезапуску
