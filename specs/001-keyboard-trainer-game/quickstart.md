# Quickstart: Дитячий клавіатурний тренажер (001-keyboard-trainer-game)

**Date**: 2026-07-18 | **Plan**: [plan.md](./plan.md)

Гід із запуску та ручної валідації. Реалізаційні деталі — у
[contracts/](./contracts/) та [data-model.md](./data-model.md).

## Передумови

- Windows 11, браузер Edge або Chrome (актуальна версія).
- Будь-який локальний статичний сервер (див. research.md R1 — `file://` не
  підтримує ES6-модулі та fetch аудіо). Інтернет НЕ потрібен.
- Аудіофайли наявні в репозиторії: `sounds/{jump,explode,victory,click}.wav`,
  `music/{menu,game,gameover,win}.mp3` (вже перевірено).

## Запуск

З кореня репозиторію — будь-який з варіантів:

```powershell
# Варіант A: Python
python -m http.server 8080

# Варіант B: VS Code — розширення Live Server → Open with Live Server
```

Відкрити `http://localhost:8080` у браузері.

## Сценарії валідації

### V1. Завантаження та меню (US5, US2)

1. Відкрити сторінку → видно неоновий екран «ЗАВАНТАЖЕННЯ...».
2. Після завантаження — головне меню: демо-кубик сам стрибає через шипи;
   після першого кліку грає `menu.mp3`; по центру гігантська кругла кнопка
   «СТАРТ», зліва «ВИБІР РІВНЯ», справа шестірня «НАЛАШТУВАННЯ».
3. Кожен клік по кнопці — звук кліку.

✅ Очікування: FR-001..FR-007; меню з'являється ≤ 5 с (SC-001).

### V2. Проходження рівня (US1)

1. Натиснути «СТАРТ» → Рівень 1, музика `game.mp3`, зверху смуга прогресу 0%.
2. На клавіатурі внизу екрана літери «А», «О» підсвічені тьмяно-зеленим;
   клавіша найближчого шипа блимає червоним/жовтим.
3. Натискати підсвічену літеру, коли кубик у зоні перед шипом → стрибок,
   звук `jump.wav`, прогрес росте, за кубиком trail.
4. Пропустити шип → вибух із частинками, `explode.wav`, музика `gameover.mp3`,
   екран поразки з % і кнопками «повторити»/«меню».
5. Пройти всі шипи → 100%, `victory.wav` + `win.mp3`, екран перемоги.

✅ Очікування: FR-016..FR-021, FR-024, FR-026; реакція на клавішу миттєва
(SC-003), анімація плавна (SC-004).

### V3. Прогресія та картки рівнів (US3)

1. З чистим прогресом (DevTools → Application → Local Storage → видалити
   `dfp_save_v1`, перезавантажити): у «ВИБІР РІВНЯ» доступний лише Рівень 1,
   рівні 2–6 сірі із замком, клік по них нічого не робить.
2. Пройти Рівень 1 на 100% → повернутися: Рівень 2 неоновий; картка Рівня 1
   показує «Кращий результат: 100% | HighScore: Y очок».
3. Перезавантажити сторінку → прогрес зберігся (SC-005, SC-006).
4. Пройти Рівень 1 повторно гірше → рекорд НЕ погіршився.

✅ Очікування: FR-012..FR-015; схема — contracts/storage-schema.md.

### V4. Складність EASY/HARD (US4)

1. Налаштування → EASY. У грі натиснути неправильну літеру задовго до шипа →
   нічого не сталося; кубик вибухає лише від фізичного контакту з шипом.
2. Налаштування → HARD. Натиснути неправильну літеру → миттєвий вибух;
   пропустити таймінг → миттєвий вибух.
3. Обрати HARD, перезавантажити сторінку → HARD збережено.

✅ Очікування: FR-008..FR-011.

### V5. Введення та edge cases

1. Перемкнути системну розкладку Windows на EN → гра однаково реагує на
   фізичні клавіші (event.code): фізична клавіша «Ф» (латинська A) працює як «Ф».
2. Під час гри натиснути Space, стрілки → сторінка не скролиться (SC-008).
3. Згорнути вікно посеред рівня на ~5 с, повернутися → кубик не
   «телепортувався» у шип (пауза/кламп delta).
4. Тимчасово перейменувати `sounds/jump.wav`, перезавантажити → гра
   запускається і працює без звуку стрибка, консоль без фатальних помилок.
   (Повернути файл після перевірки.)
5. Змінити розмір вікна → поле і клавіатура адаптуються без спотворень.

✅ Очікування: FR-002, FR-022, FR-023, FR-027, FR-029.

### V6. Фони за рівнями

- Рівні 1–2: рухома темно-синя неонова сітка (паралакс).
- Рівні 3–4: далекі фіолетові хмарочоси/гори із сяючими вікнами.
- Рівні 5–6: пульсуюче рожево-бірюзове тло, еквалайзер у такт, світлові хвилі
  від правильних натискань; на Рівні 6 шипи йдуть ритмічними групами.

✅ Очікування: FR-017, FR-025.

## Критерій завершення

Усі сценарії V1–V6 проходять без помилок у консолі браузера; вимоги
Конституції (Принципи I–VI) дотримані — жодних зовнішніх запитів
(вкладка Network показує лише localhost).
