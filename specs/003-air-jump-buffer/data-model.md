# Data Model: Буферизація стрибка (003-air-jump-buffer)

**Date**: 2026-07-18 | **Plan**: [plan.md](./plan.md)

Одна нова runtime-сутність усередині класу `Engine`. Персистентних даних немає
(FR-041).

## 1. JumpBuffer (внутрішнє поле Engine)

| Поле | Тип | Опис |
|------|-----|------|
| `jumpBuffer` | Spike \| null | Посилання на конкретний шип (research.md R1), для якого гравець натиснув правильну літеру в повітрі; `null` = буфер порожній |

**Ініціалізація**: `null` у `reset()` разом з іншими змінними забігу
(вимога користувача №4).

**Серіалізація**: відсутня — поле не потрапляє в `localStorage` (FR-041).

## 2. Життєвий цикл (переходи станів буфера)

```text
                     ┌──────────────────────────────────────────┐
                     │  правильна літера, onGround === false    │
   null ────────────►│  jumpBuffer = spike (FR-037)             │
    ▲                └──────────────────────────────────────────┘
    │                          │ нове натискання в повітрі:
    │                          │ перезапис (FR-039)
    │                          ▼
    │                ┌──────────────────┐
    │                │ jumpBuffer=spike'│
    │                └──────────────────┘
    │                          │ перехід onGround: false → true
    │                          ▼
    │        ┌─────────────────────────────────────────────┐
    │        │ spike.state === 'ahead' і 0 < gap ≤ okPx ?  │
    │        └─────────────────────────────────────────────┘
    │              так │                        │ ні
    │                  ▼                        ▼
    │        авто-стрибок (FR-038):     тихе скасування
    │        cleared + очки + jump()    (шип пройдено або
    │                  │                 недосяжний)
    └──────────────────┴────────────────────────┘
                       також → null при:
                       explode() — вибух будь-якої причини (FR-040а,б)
                       reset()  — старт/рестарт забігу (FR-041)
```

## 3. Інваріанти

1. Буфер ніколи не виконує стрибок для шипа зі станом `cleared` або `hit`
   (перевірка `state === "ahead"` — FR-040).
2. Одночасно існує щонайбільше один забуферений намір (одне поле — FR-039).
3. Автостибок виконується лише в кадрі приземлення (перехід
   `false → true`), ніколи — доки кубик у повітрі чи стоїть на землі
   (research.md R3).
4. Поведінка `handleLetter` для `demoMode`, помилкових літер та натискань на
   землі — байт-у-байт як у фічі 001 (FR-042, FR-043).
5. `jumpBuffer` не впливає на `getState()`, `getOutcome()`,
   `getTargetLetter()`, збереження та рендеринг.
