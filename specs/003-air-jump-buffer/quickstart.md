# Quickstart: Буферизація стрибка (003-air-jump-buffer)

**Date**: 2026-07-18 | **Plan**: [plan.md](./plan.md)

## Передумови

- Фічі 001–002 реалізовані; гра запускається:
  `python -m http.server 8080` → `http://localhost:8080`.
- Для сценаріїв HARD: Налаштування → HARD. Для швидкого доступу до Рівня 6
  можна тимчасово пройти рівні або перевіряти на Рівні 5 (теж щільний).

## Сценарії валідації

### V12. Буферизований стрибок у щільному ланцюжку (US1)

1. Запустити Рівень 5 або 6 (EASY). Дійти до групи щільних шипів.
2. Перелетівши перший шип, ЩЕ В ПОЛЬОТІ натиснути літеру наступного шипа.
3. Очікування: одразу після приземлення кубик стрибає сам (звук стрибка,
   очки нараховано), шип перелетіло без вибуху.
4. Повторити з двома натисканнями різних літер у одному польоті —
   виконується лише останнє.

✅ Очікування: FR-037, FR-038, FR-039; SC-013, SC-015.

### V13. Чесність HARD (US2)

1. Увімкнути HARD, запустити рівень.
2. У польоті над шипом натиснути ПРАВИЛЬНУ літеру наступного шипа →
   вибуху немає; після приземлення — автостибок.
3. У польоті натиснути НЕПРАВИЛЬНУ літеру → миттєвий вибух (як раніше).
4. На землі поза зоною натиснути правильну літеру → миттєвий вибух
   (незмінна поведінка HARD).

✅ Очікування: FR-037, FR-042, FR-043; SC-014.

### V14. Цілісність буфера (US3)

1. У польоті натиснути правильну літеру наступного шипа і НАВМИСНО
   врізатись у поточний шип (натиснути пізно) → вибух; після
   «СПРОБУВАТИ ЩЕ» (або Enter, фіча 002) новий забіг стартує БЕЗ
   самовільного стрибка на першому кроці.
2. У польоті натиснути літеру шипа, що буде пройдений до приземлення
   (натиснути в момент прольоту над ним) → після приземлення автостибка
   для нього немає.
3. Натиснути правильну літеру дуже рано у довгому польоті, коли наступний
   шип ще далеко → після приземлення стрибка немає, покарання немає
   (буфер тихо скасовано); повторне натискання у звичайній зоні працює.

✅ Очікування: FR-040, FR-041; SC-016.

### V15. Регресія фіч 001–002

1. Звичайне проходження Рівня 1 (усі натискання на землі) — поведінка,
   очки, звуки без змін (quickstart 001, V2).
2. EASY: помилкові натискання ігноруються; HARD: помилка → вибух (V13.3).
3. Демо-заставка меню грає сама без збоїв ≥ 60 с.
4. Enter на екранах завершення перезапускає рівень (quickstart 002, V7–V8).
5. Прогрес/рекорди зберігаються та відновлюються (quickstart 001, V3);
   у DevTools → Application → Local Storage значення `dfp_save_v1` не
   містить жодних полів буфера.

✅ Очікування: незмінність наявної поведінки; FR-041.

## Критерій завершення

V12–V15 проходять; консоль чиста; headless-тести Engine (див. tasks.md)
зелені; Network — лише localhost.
