# Quickstart: Валідація системи балів, досягнень та UI нагород

**Дата**: 2026-07-20

## Передумови

1. Запущений локальний HTTP-сервер: `start-http.cmd` (або `npx http-server -p 8080`)
2. Відкритий браузер на `http://localhost:8080` (Chrome/Edge рекомендовано для DevTools)
3. Відкриті DevTools (F12) → Console + Application → Local Storage
4. Усі аудіофайли в `music/` та `sounds/` завантажені

---

## Сценарій 1: Перевірка нової формули балів (P1)

**Мета**: Переконатися, що бали нараховуються за новою адитивною формулою.

### Кроки

1. Відкрити гру, натиснути «ГРАТИ»
2. Обрати будь-який рівень (наприклад, 1-1)
3. У налаштуваннях (кнопка «⚙») встановити:
   - Difficulty: **EASY**
   - Hit Window: **NORMAL**
   - Speed: **NORMAL**
4. Розпочати рівень, дочекатися першого шипа
5. Натиснути правильну літеру в момент, коли кубик над шипом
6. Перевірити в консолі або на progress bar: **рахунок збільшився на 120** (100 база + 0 + 20 + 0)
7. Змінити налаштування (не виходячи з рівня):
   - Рестарт → Settings → HARD / NORMAL / FAST
8. Повторити — **рахунок має збільшуватися на 210** за кожне OK-влучання
9. Перевірити EASY / LARGE / SLOW — влучання в іншу зону (не OK) → **+60 балів**

### Очікувані результати

| Конфігурація | OK зона | Інша зона |
|-------------|---------|-----------|
| EASY / NORMAL / NORMAL | 120 | 100 |
| HARD / NORMAL / FAST | 210 | 190 |
| EASY / LARGE / SLOW | 80 | 60 |
| HARD / LARGE / SLOW | 130 | 110 |

## Сценарій 2: Перевірка досягнення perfectEasy (P2)

**Мета**: Отримати Срібний Максимум на рівні 1-1.

### Кроки

1. Відкрити DevTools → Application → Local Storage → `dfp_save_v1`
2. Переконатися, що рівень 1-1 має `perfect: null`
3. Запустити рівень 1-1 із налаштуваннями: **EASY / NORMAL / NORMAL**
4. Пройти рівень, влучаючи в OK кожного разу (maxEasy = spikeCount × 120)
5. На екрані VICTORY: фінальний рахунок має дорівнювати maxEasy
6. Перевірити localStorage: `levels["1"].perfect === "easy"`
7. Повернутися в меню вибору рівнів → картка рівня 1-1 має **срібну рамку та срібну зірку ★**
8. Перезавантажити сторінку → срібне оформлення зберігається

### Очікувані результати

- `localStorage["dfp_save_v1"].progress.levels["1"].perfect === "easy"`
- Картка рівня 1-1 має клас `.perfect-silver`
- Срібна зірка ★ над номером рівня з металевим градієнтом

## Сценарій 3: Перевірка досягнення perfectHard та пріоритету (P2)

**Мета**: Отримати Золотий Максимум, перевірити що він перекриває Срібний.

### Кроки

1. Очистити localStorage або переконатися, що рівень 1-1 має `perfect: null`
2. Запустити рівень 1-1 із налаштуваннями: **HARD / NORMAL / FAST**
3. Пройти рівень, влучаючи в OK кожного разу (maxHard = spikeCount × 210)
4. Перевірити localStorage: `levels["1"].perfect === "hard"`
5. Картка рівня 1-1 має **золоту рамку з пульсацією та золоту зірку ★**
6. Тепер пройти рівень 1-1 на EASY з ідеальним результатом
7. Перевірити localStorage: `perfect` **залишається "hard"** (золото не втрачається)

### Очікувані результати

- `localStorage["dfp_save_v1"].progress.levels["1"].perfect === "hard"`
- Картка рівня має клас `.perfect-gold` з анімацією `goldPulse`
- Після perfectEasy на EASY статус не знижується

## Сценарій 4: Золоте свічення кубика на Canvas (P4)

**Мета**: Побачити золотий glow навколо кубика під час гри на рівні з perfectHard.

### Кроки

1. Встановити `perfect: "hard"` для рівня 1-1 вручну через DevTools:
   ```js
   const save = JSON.parse(localStorage.getItem("dfp_save_v1"));
   save.progress.levels["1"].perfect = "hard";
   localStorage.setItem("dfp_save_v1", JSON.stringify(save));
   ```
2. Перезавантажити сторінку
3. Запустити рівень 1-1
4. Спостерігати кубик: **навколо нього золоте свічення** (жовтий shadowBlur)
5. Виконати стрибок — за кубиком має тягнутися **золотавий шлейф** із частинок
6. Перевірити FPS (через DevTools → Performance або візуально) — не нижче 55 FPS

### Очікувані результати

- Кубик має яскраве золоте свічення (`shadowColor: '#ffaa00'`, `shadowBlur: 18`)
- Під час стрибків видно золоті частинки шлейфу
- FPS стабільний (≥55 на інтегрованій графіці)

## Сценарій 5: Срібний контур кубика (P4)

**Мета**: Побачити срібний контур навколо кубика для perfectEasy.

### Кроки

1. Встановити `perfect: "easy"` для рівня 1-2 через DevTools
2. Запустити рівень 1-2
3. Спостерігати кубик: **чіткий сріблясто-білий контур** навколо скіна
4. Контур обертається разом із кубиком під час стрибків

### Очікувані результати

- Тонкий срібний контур (`strokeStyle: '#d4dce8'`, `lineWidth: 1.8`)
- Контур синхронізований із обертанням кубика
- Немає золотого свічення чи шлейфу

## Сценарій 6: Картки скінів зі статусами (P4)

**Мета**: Перевірити срібні/золоті контури на картках скінів у меню вибору.

### Кроки

1. Встановити `perfect: "hard"` для рівня 1-3 через DevTools
2. Встановити `perfect: "easy"` для рівня 1-2 через DevTools
3. Перезавантажити сторінку
4. На головному меню натиснути на іконку скіна (відкрити модальне вікно скінів)
5. Картка скіна рівня 1-2 має **срібний контур** (`.perfect-silver`)
6. Картка скіна рівня 1-3 має **золоту рамку зі свіченням** (`.perfect-gold`)
7. Закриті скіни без досягнень — стандартний вигляд

### Очікувані результати

- `.skin-card.perfect-silver` має срібний `border-color` та `box-shadow`
- `.skin-card.perfect-gold` має золотий `border-color` та анімацію `goldPulse`
- Звичайні скіни без змін

## Сценарій 7: Зворотна сумісність localStorage (Edge Case)

**Мета**: Переконатися, що старі збереження без поля `perfect` не ламаються.

### Кроки

1. Зберегти поточний прогрес
2. Видалити поле `perfect` з усіх рівнів у localStorage:
   ```js
   const save = JSON.parse(localStorage.getItem("dfp_save_v1"));
   for (const key of Object.keys(save.progress.levels)) {
       delete save.progress.levels[key].perfect;
   }
   localStorage.setItem("dfp_save_v1", JSON.stringify(save));
   ```
3. Перезавантажити сторінку
4. Гра запускається без помилок
5. Перевірити localStorage: усі рівні отримали `perfect: null`
6. Усі картки рівнів — стандартний вигляд, без зірок
7. Гра працює, нові досягнення можна отримувати

### Очікувані результати

- `sanitizeSaveData()` додає `perfect: null` для рівнів без цього поля
- Жодних помилок у консолі
- Існуючий прогрес (bestPct, highScore) збережено

## Сценарій 8: Відображення maxScore на progress bar (SC-001)

**Мета**: Підтвердити, що гравець бачить поточний рахунок і максимум.

### Кроки

1. Запустити будь-який рівень
2. Подивитися на правий бік progress bar
3. Текст має формат: **«Очки: 240 / 1680»** (поточний / максимум)
4. Максимум змінюється залежно від конфігурації (EASY vs HARD)

### Очікувані результати

- Формат: `Очки: {score} / {maxForMode}`
- Для EASY: `maxForMode = this.maxEasy`
- Для HARD: `maxForMode = this.maxHard`
