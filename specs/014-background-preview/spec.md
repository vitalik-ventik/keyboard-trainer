# Feature Specification: Сторінка попереднього перегляду фонів

**Feature Branch**: `014-background-preview`

**Created**: 2026-07-19

**Status**: Draft

**Input**: User description: "Потрібно зробити preview.html файл, який буде показувати список всіх левелів (80% по висоті і ширині від робочої області) і буде генерувати бекграунд в них."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Перегляд усіх фонів в одному місці (Priority: P1)

Розробник хоче швидко оцінити, як виглядають усі 10 типів динамічних фонів на різних рівнях, не запускаючи гру. Він відкриває `preview.html` у браузері і бачить сітку з карток — кожна картка показує назву рівня, його лігу та анімований фон цього рівня. Картки займають 80% ширини та висоти робочої області, автоматично масштабуючись під розмір вікна.

**Why this priority**: Це єдина функція файлу — без неї файл не має сенсу.

**Independent Test**: Відкрити `preview.html` у браузері — має відобразитись сітка карток з анімованими фонами для всіх рівнів.

**Acceptance Scenarios**:

1. **Given** розробник відкриває `preview.html`, **When** сторінка завантажується, **Then** на екрані відображається сітка з картками, кожна з яких показує назву рівня та анімований фон.
2. **Given** відкрито `preview.html`, **When** розробник змінює розмір вікна браузера, **Then** сітка карток перебудовується, займаючи ~80% ширини та висоти вікна.

### Edge Cases

- Що відбувається, якщо `engine.js` або `backgrounds.js` не завантажились? Сторінка має показати повідомлення про помилку замість пустої сторінки.
- Як поводиться сторінка на дуже малих екранах (< 800px ширини)? Картки мають зменшуватись, але залишатись впізнаваними.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: Сторінка MUST відображати сітку карток для всіх 31 рівня з `LEVELS_CONFIG`.
- **FR-002**: Кожна картка MUST показувати назву рівня, назву ліги та безперервно анімований фон із використанням `BackgroundRenderer`.
- **FR-003**: Сітка карток MUST займати 80% ширини та 80% висоти вікна браузера.
- **FR-004**: Сторінка MUST бути самодостатньою (відкривається локально, без сервера).
- **FR-005**: Сторінка MUST коректно реагувати на зміну розміру вікна (адаптивна сітка).

### Key Entities

- **Картка рівня (Level Card)**: Візуальний блок, що містить canvas-елемент із рендером фону, назву рівня та назву ліги.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: Усі 31 рівень відображаються на сторінці з відповідним анімованим фоном.
- **SC-002**: Сторінка завантажується та починає анімацію протягом 2 секунд після відкриття.
- **SC-003**: Сітка коректно перебудовується при зміні розміру вікна без втрати карток.

## Assumptions

- `js/backgrounds.js` та `js/engine.js` доступні в тій самій директорії, що й `preview.html`.
- Файл відкривається локально через `file://` протокол (ES6 модулі працюють через `type="module"`).
- Сторінка не потребує звуків, музики чи логіки введення — лише візуальний перегляд.
