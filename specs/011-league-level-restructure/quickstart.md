# Quickstart: Валідація ліг та рівнів

**Feature**: 011-league-level-restructure
**Date**: 2026-07-19

## Prerequisites

- Запущений локальний HTTP-сервер: виконати `start-http.cmd` у корені проекту
- Відкрити `http://localhost:8080` (або інший порт згідно з сервером) у Chrome/Edge/Firefox
- Переконатися, що всі звукові файли (`sounds/*.wav`, `music/*.mp3`) присутні

## Validation Scenarios

### Сценарій 1: Перемикання вкладок ліг

1. На головному меню натиснути «ВИБІР РІВНЯ»
2. **Очікується**: Відображаються 5 вкладок: [Базовий] [Середній] [Складний] [Майстер] [Бос]
3. Вкладка «Базовий» активна за замовчуванням, показано сітку з 16 карток
4. Перемкнути на «Середній» → показано 8 карток
5. Перемкнути на «Складний» → 4 картки
6. Перемкнути на «Майстер» → 2 картки
7. Перемкнути на «Бос» → 1 картка «ФІНАЛЬНИЙ ДЕМОН»
8. Повернутися на «Базовий» → кількість карток не змінилася

### Сценарій 2: Картка рівня — вміст

1. На вкладці «Базовий» перевірити картку 1-1
2. **Очікується**: Заголовок «1-1: Перші кроки» по центру картки
3. Внизу картки — блок з напівпрозорим текстом «а о в л» (або через кому)
4. Картка інтерактивна (доступна для кліку)
5. Усі інші картки (1-2 .. 1-16) мають сірий вигляд (заблоковані)

### Сценарій 3: Прогресивне розблокування

1. Очистити localStorage: `localStorage.clear()` у консолі
2. Оновити сторінку
3. На вкладці «Базовий» — лише 1-1 доступний. Запустити 1-1.
4. Пройти рівень до кінця (досягти фінішу)
5. **Очікується**: Екран перемоги з кнопкою «НАСТУПНИЙ РІВЕНЬ»
6. Натиснути «НАСТУПНИЙ РІВЕНЬ» → запущено 1-2
7. Або натиснути «ДО МЕНЮ» → «ВИБІР РІВНЯ» → 1-1 та 1-2 тепер доступні
8. Оновити сторінку — 1-1 та 1-2 залишаються доступними

### Сценарій 4: HUD під час гри

1. Запустити рівень 1-1
2. **Очікується**: У верхній частині Canvas відображається:
   ```
   Ліга: Базова | 1-1: Перші кроки
   ```
3. Запустити рівень 5-1 (потрібно спочатку розблокувати через `localStorage` або пройти всі рівні)
4. **Очікується**: HUD показує:
   ```
   Ліга: Бос | 5-1: ФІНАЛЬНИЙ ДЕМОН
   ```

### Сценарій 5: Прев'ю літер на різних рівнях

1. Перевірити 1-1 → прев'ю: «а о в л»
2. Перевірити 1-5 («Ліва вертикаль») → прев'ю: «й ц ф і я ч» (6 літер)
3. Перевірити 2-2 («Дах клавіатури») → прев'ю: «й ц у к е н г ш щ з х ї» (12 літер)
4. Перевірити 5-1 («ФІНАЛЬНИЙ ДЕМОН») → прев'ю: «Усі 33 літери» (або повний список)

### Сценарій 6: Адаптивність табів

1. Зменшити розмір вікна браузера до ~400px ширини
2. **Очікується**: Вкладки ліг або зменшуються, або переносяться — усі 5 видимі
3. Розгорнути на повний екран → вкладки на одному рядку, нормальний розмір

### Сценарій 7: Стійкість до помилок localStorage

1. Відкрити консоль браузера (F12)
2. Виконати: `localStorage.clear()`
3. Оновити сторінку
4. **Очікується**: Гра завантажується без помилок у консолі
5. У виборі рівнів — лише перші рівні кожної ліги доступні
6. Запустити доступний рівень → гра працює коректно

### Сценарій 8: Фони рівнів

1. Запустити 1-1 → темно-синя неонова сітка з паралаксом
2. Запустити 1-2 → темно-фіолетовий градієнт із силуетами хмарочосів
3. Запустити 2-2 → малиновий Synthwave із сонцем
4. Запустити 5-1 → чорне тло, криваво-червоні неонові лінії, хаотична пульсація

### Сценарій 9: Зворотна сумісність (старі збереження)

1. Якщо є старі дані з попередньої версії:
   - `localStorage.setItem("dfp_save_v1", JSON.stringify({..., progress: { unlocked: 7, levels: {...} }}))`
2. Оновити сторінку
3. **Очікується**: Рівні 1–7 доступні в Базовій лізі (для старих збережень unlocked ≤ 16 залишається в лізі 1, unlocked > 16 — мапіться на відповідну лігу)

## Notes

- Для прискореного тестування розблокування можна встановити `progress.unlocked` вручну через консоль
- Усі перевірки виконуються без додаткових інструментів — лише браузер і консоль розробника
