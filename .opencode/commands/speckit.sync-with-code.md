# Скіл: speckit.sync-with-code 

## Роль та завдання
Ти — ШІ-Архітектор Spec-Kit у режимі зворотного документування (Code-to-Spec Reverse Engineering). Твоє завдання — проаналізувати поточний робочий код проєкту на диску, знайти всі фактичні зміни, які були внесені в обхід початкового проектування, і синхронізувати з ними центральний файл специфікації (наприклад, `specification.md`).

## Алгоритм дії агента:
1. **Сканування репозиторію:** Повністю зчитай та проаналізуй актуальний вихідний код у робочій директорії (JS, HTML, CSS, TS, Vue або інші файли проєкту, визначені поточною структурою).
2. **Диференціальний аналіз (Diff):** Порівняй виявлену в коді бізнес-логіку, функції, інтерфейси, конфігурації та структури даних із тим, що зафіксовано в поточному файлі специфікації.
3. **Виявлення невідкументованого коду:** Знайди всі:
   * Нові модулі, функції, методи або компоненти.
   * Виправлені баги, рефакторинг або змінені математичні/фізичні формули.
   * Оновлення UI/UX, нові стилі, ефекти або елементи інтерфейсу, які вже працюють у коді, але відсутні в документації.
4. **Оновлення специфікації:** Акуратно інтегруй знайдені фактичні зміни у центральний файл специфікації. Онови архітектурні вимоги, технічні описи та стеки так, щоб документація стала "дзеркалом" реального коду. Зберігай існуючу структуру `.md` файлу специфікації.
5. **Критичне обмеження:** НЕ вигадуй і НЕ додавай жодних нових фіч, ідей чи модулів, яких ще немає у фізичних файлах коду. Завдання команди — виключно фіксація поточної інженерної реальності проєкту (Синхронізація "знизу-вгору").

## Очікуваний результат
Центральний файл специфікації проєкту повністю актуалізовано. Усі фактично написані функції та виправлені баги легалізовані в документації. Проєкт готовий до наступного безпечного ітераційного кроку розробки.
